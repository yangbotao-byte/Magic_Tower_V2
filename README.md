# 魔塔 C++ — 魔塔之旅

> 一款基于 **EasyX** 图形库的 2D 横版动作角色扮演游戏（ARPG），使用 C++ 编写，Visual Studio 2022 构建。

---

## 🎮 项目简介

**魔塔之旅** 是一款带有剧情驱动的 2D 动作小游戏。玩家将操控勇者，穿越雨夜、踏足地图、挑战怪物，最终揭开魔塔的秘密。

游戏采用逐帧动画系统、粒子特效、BGM 音效，实现了完整的游戏循环：**欢迎界面 → 加载 → 主世界 → 地图选关 → 关卡战斗 → 结局字幕**。

## ✨ 已实现功能

### 核心系统
| 模块 | 说明 |
|------|------|
| 🎬 **欢迎界面** | 星空粒子背景 + 火焰粒子特效 + 彩虹色菜单（HSV 渐变），支持键盘上下选择 |
| 📦 **资源加载** | 带进度条的异步加载界面，逐帧加载英雄动画资源 |
| 🌧️ **主世界** | 三层视差滚动背景（远景/中景/近景），实时天气系统（雨滴 + 闪电 + 水面溅射特效），BGM 音效 |
| 🗺️ **地图选关** | 基于图的节点地图，左键 Dijkstra 最短路径移动，右键 Kruskal 最小生成树可视化，Enter 确认选关 |
| ⚔️ **关卡战斗** | 横版战斗场景，英雄 vs 敌人，HP 血条显示，攻击判定框系统 |
| 🎉 **结局字幕** | 击败敌人后播放纵向滚动字幕（制作人员名单 + 致谢） |
| 🔊 **音量设置** | 可视化音量滑块，鼠标拖拽调节，ESC 返回 |

### 角色系统
| 功能 | 说明 |
|------|------|
| 🧍 **站立动画** | 9 帧待机动画（`hero/stand/`） |
| 🚶 **行走动画** | 10 帧行走动画（`hero/walk/`），支持左右镜像 |
| ⚔️ **攻击动画** | 16 帧攻击动画（`hero/attack/`），支持左右方向 |
| 🦘 **跳跃动画** | 4 帧跳跃动画（`hero/jump/`），含重力模拟 |
| ❤️ **HP 血条** | 实时血量显示，受伤/冰冻机制 |

### 敌人 AI 系统
| 功能 | 说明 |
|------|------|
| 👣 **追踪移动** | 敌人自动追踪玩家位置 |
| 🔥 **火焰技能** | 释放火焰粒子，造成 15 点伤害 |
| ❄️ **冰冻技能** | 释放冰霜粒子，冻结玩家 30 帧 |
| 💀 **死亡判定** | HP 归零后移除，触发胜利条件 |

### 粒子特效
- 🔥 火焰粒子（上升 + 随机飘散 + 颜色衰减）
- ❄️ 冰霜粒子（扩散 + 半透明效果）
- 🌟 星空粒子（欢迎界面背景）
- 💧 雨滴溅射（主世界天气）

## 🚀 构建与运行

### 环境要求

1. **Visual Studio 2022**（或其他支持 v143 工具集的 VS 版本）
2. **[EasyX 图形库](https://easyx.cn/)**（下载安装后自动集成到 VS）
3. **Windows 10/11**

### 构建步骤

1. 克隆仓库：
   ```bash
   git clone https://github.com/你的用户名/魔塔C-.git
   ```

2. 用 Visual Studio 打开 `魔塔C++/魔塔C++.vcxproj`

3. 选择配置：**Debug x64** 或 **Release x64**

4. 按 `F5` 编译运行

## ⚠️ 注意事项

### 1. 工作目录路径问题（重要！）

代码中所有资源加载都使用**相对路径**（如 `./hero/stand/frame_0.png`），默认的 VS 工作目录是项目根目录（`.vcxproj` 所在目录），即 **`魔塔C++/`**。

如果运行时提示找不到文件，请检查：

- **VS 中设置工作目录**：项目属性 → 调试 → 工作目录 → 设为 `$(ProjectDir)`
  - `$(ProjectDir)` = `魔塔C++/`（.vcxproj 所在目录）

- **或者直接在代码中修改**：将所有 `./xxx/` 路径改为你的实际绝对路径，例如：
  ```cpp
  // 原来的相对路径
  sprintf(path, "./hero/stand/frame_%d.png", i);
  // 改为绝对路径（示例）
  sprintf(path, "D:/porject/魔塔C++/魔塔C++/hero/stand/frame_%d.png", i);
  ```

### 2. EasyX 库安装

EasyX 不是 C++ 标准库，需要单独安装：
- 前往 [https://easyx.cn/](https://easyx.cn/) 下载安装包
- 安装时会自动检测 VS 版本并完成配置
- 如果编译报 `cannot open 'graphics.h'`，说明 EasyX 未安装或未正确配置

### 3. 字符编码

项目源码使用 **GBK/GB2312** 编码（中文 Windows 默认）。如果在非中文系统或 VS Code 中打开出现乱码：
- VS：文件 → 高级保存选项 → 选择 "简体中文 (GB2312)"
- VS Code：右下角点击编码 → "通过编码重新打开" → 选择 "GB 2312" 或 "GBK"

### 4. 音频文件

背景音乐依赖 `music/` 目录下的 `.mp3` 文件：
- `music/begin.mp3` — 主界面/地图 BGM
- `music/111.mp3` — 闪电音效

如果缺少音频文件，游戏仍可运行，只是没有背景音乐。如需移除音频功能，注释 `main.cpp` 中的 `mciSendString` 调用即可。

### 5. Debug vs Release

- **Debug x64**：字符集设为 MultiByte（兼容中文路径和输出）
- **Release x64**：字符集为 Unicode，打包分发时使用
- Win32 配置未充分测试，建议使用 **x64**

### 6. 已知问题

- `垃圾代码.cpp` 是废弃/实验代码，与主流程无关，可忽略
- 英雄跳跃动画 `updatejump()` 已注释掉，当前跳跃逻辑在 `主世界.cpp` 和 `关卡1.cpp` 中直接实现
- 敌人路径 `./e_m_001/` 下的动画帧序号从 `0000` 到 `0007`，与 `run_2.json` 骨骼配置配套
- `主世界.cpp` 中开门触发区域硬编码为 `x > 810 && x < 910`，如需调整请修改对应数值
